# 🤖 CHATBOT.md — Module: ผู้ช่วย AI "น้องพอร์ช"

> ผู้ช่วยถาม-ตอบข้อมูลในระบบ (ราคา/สต็อก/DOT/cipher) · ปุ่มหุ่นยนต์ลอยมุมขวาล่าง · **เคารพสิทธิ์ตาม role เสมอ**
> Prototype: `app/chatbot.jsx` — โชว์พฤติกรรม UX จริง (NLU เป็น rule-based mock)

---

## 1. ภาพรวม
- **ทำอะไร:** พิมพ์คำถามภาษาธรรมชาติ (ไทย/อังกฤษ) แล้วได้คำตอบ + การ์ดสินค้าที่เกี่ยวข้องในแชท
- **อยู่ที่ไหน:** ปุ่มกลมสีเหลืองรูปหุ่นยนต์ มุมขวาล่างทุกหน้า (มีจุดเขียว = online) → กดเปิดแผงแชท
- **ใครใช้ได้:** ทุก role — แต่ **คำตอบกรองข้อมูลตามสิทธิ์** (เช่น เซลล์ยางใหญ่ไม่เห็นทุน)

---

## 2. Intent ที่รองรับ (Phase 1)

| ผู้ใช้พิมพ์ (ตัวอย่าง) | Intent | ตอบกลับ |
|---|---|---|
| "ราคา 215/70R15", "michelin", "21515mk" | **product_search** | การ์ดสินค้า (ราคา/สต็อก/DOT ตามสิทธิ์) |
| "ยาง DOT เก่า", "ค้างสต็อก" | **old_dot** | ยาง DOT ≥2 ปี ที่ควรเร่งระบาย |
| "ยางใกล้หมด", "ต้องเบิก", "low stock" | **low_stock** | รายการสต็อก <30 หรือสถานะ `+` |
| "ยางเก๋งทั้งหมด", "แบตเตอรี่" | **category** | สรุปจำนวน + ตัวอย่างในหมวด |
| "clearance", "เคลียร์" | **clearance** | รายการสถานะ `C` |
| "รหัสลับคืออะไร" | **cipher_info** | อธิบาย cipher (เฉพาะคนมีสิทธิ์) |
| "ช่วยอะไรได้บ้าง", "help" | **help** | รายการความสามารถ |
| "สวัสดี" | **greeting** | ทักทาย |
| (ไม่ตรง) | **fallback** | แนะนำให้ลองพิมพ์ขนาด/ยี่ห้อ |

> **Quick chips:** มีปุ่มคำถามแนะนำใต้แชท (ราคา / ยางใกล้หมด / DOT เก่า / ยางเก๋ง) กดถามได้ทันที

## 3. ⚠️ ความปลอดภัย — กรองตาม role (สำคัญที่สุด)

การ์ดผลลัพธ์ในแชท **ใช้กฎเดียวกับตาราง** (`COL_PERMS[role]`):

| role | เห็นในการ์ดแชท |
|---|---|
| 🔧 Admin | ราคาขาย + ทุน(รหัส) + สต็อก + DOT |
| 🛞 เซลล์ยางใหญ่ | ราคาขาย + สต็อก + DOT (ไม่มีทุน/ส่ง) |
| 🏪 เซลล์ดูแลร้านค้า | รหัสส่ง B/A/S + สต็อก + DOT (ไม่มีราคาขาย/ทุน) |
| 🛒 พนักงานหน้าร้าน | ราคาขาย + รหัสส่ง + สต็อก |

> 🔒 **โปรดทำที่ฝั่ง server เหมือนกัน:** chatbot ต้องเรียก endpoint ที่กรองคอลัมน์ตาม role อยู่แล้ว (ดู `API.md §2`) — **ห้าม**ให้ LLM เห็นเลขทุนของ role ที่ไม่มีสิทธิ์ ตั้งแต่ใน context

## 4. การ implement จริง (Laravel)

Prototype ใช้ rule-based เพื่อโชว์ UX — production ทำได้ 2 ระดับ:

### ระดับ A — Intent + Query (เริ่มที่นี่ก่อน, ถูก/เร็ว/คุมได้)
```
POST /api/chatbot/ask   { message, context? }
```
1. classify intent (regex/keyword หรือ LLM เล็ก)
2. ดึงข้อมูลจาก DB ผ่าน **service เดิม** (SearchService ฯลฯ) — ได้สิทธิ์ตาม role ฟรี
3. format เป็นข้อความ + rows ส่งกลับ
- ✅ ข้อมูลแม่นยำ 100% (มาจาก DB ตรง) · ไม่มี hallucinate ราคา
- ✅ ไม่ต้องส่งราคาเข้า LLM ก็ได้ (LLM แค่ช่วยเข้าใจคำถาม)

### ระดับ B — RAG / Tool-calling (Phase 3, ถ้าต้องตอบซับซ้อน)
- ใช้ **พอร์ช (Mac Mini M4)** เป็น local LLM gateway (ผ่าน AI Agent Gateway, LAN เท่านั้น — ดู `API.md §10`)
- ให้ LLM เรียก "tools" (= endpoint ที่กรอง role แล้ว) แทนการป้อนข้อมูลดิบ
- เก็บ embedding (pgvector) เฉพาะข้อมูลที่ไม่อ่อนไหว (ชื่อรุ่น/สเปก) — **ห้าม** embed ราคาทุน
- audit ทุกคำถาม-คำตอบ (category `chatbot`)

> แนะนำเริ่ม **ระดับ A** ให้ใช้งานได้จริงใน Phase 1 แล้วค่อยยกระดับ

## 5. UX details (จาก prototype)
- ปุ่ม FAB: 58px กลม สีเหลือง + เงา + จุด online เขียว · กด → สลับเป็นกากบาท
- แผงแชท: 372×560px ลอยเหนือปุ่ม · header แสดงชื่อ + สถานะ + badge role ปัจจุบัน
- ฟอง: ผู้ใช้ = เหลือง (ขวา) · บอท = surface เทา (ซ้าย) · มี typing indicator (จุดเด้ง)
- การ์ดสินค้าฝังในฟองบอท: ขนาด/ยี่ห้อ + ราคา/รหัส + สต็อก(สีตามจำนวน) + DOT(สีตามอายุ)
- รองรับ TH/EN (สลับตามภาษาแอป) · responsive (มือถือเต็มความกว้าง - margin)
